# Codebase Concerns

**Analysis Date:** 2026-05-18

---

## Critical Gaps (blocking for launch)

**Sanity CMS not wired up — all content is hardcoded:**
- Sanity packages are installed (`sanity`, `next-sanity`, `@sanity/image-url`, `@portabletext/react`) but zero Sanity configuration files exist in `src/`.
- No `sanity.config.ts`, no `src/app/studio/` route, no `src/lib/sanity.ts` client, no schema definitions anywhere.
- All property data lives entirely in `src/lib/data.ts` as a static TypeScript export (50 hardcoded properties). The core product requirement — "der Genossenschaftsvorstand kann alle Inhalte selbstständig und ohne Programmierkenntnisse pflegen" — is completely unmet.
- Files: `src/lib/data.ts`, `src/app/wohnen/page.tsx`

**Next.js version violates CLAUDE.md constraint (Next.js 15 required, 16 installed):**
- `package.json` declares `"next": "16.2.6"` and `"eslint-config-next": "16.2.6"`.
- CLAUDE.md explicitly mandates Next.js 15.x, warns that Next.js 16 + `SanityLive` causes 4–10x API request spikes, and links to the Sanity warning doc.
- File: `fewog-app/package.json`

**No `<SanityLive />` component — live content updates impossible:**
- Even after a Sanity client is wired up, `src/app/layout.tsx` has no `<SanityLive />`. Without it, content changes in Studio will never appear on the live site without a full redeploy.
- File: `src/app/layout.tsx`

**Impressum, Datenschutz, and AGB pages are missing:**
- The footer at `src/components/footer.tsx` renders "Impressum · Datenschutz · AGB" links, but those `<a>` tags have no `href` and no corresponding routes exist under `src/app/`.
- As a public German institution, a missing Datenschutzerklärung is a DSGVO violation and a missing Impressum violates §5 TMG. Both are legal requirements, not optional.

## High Priority (should fix before launch)

**Hero image is a third-party Unsplash URL — not FEWOG-owned content:**
- `src/app/page.tsx` line 74 uses `https://images.unsplash.com/photo-1502672260266-1c1ef2d93688?w=1200&q=80` as the hero image. This is a stock photo with no verified license for commercial brand use.
- Needs to be replaced with real FEWOG photography stored in Sanity's media library.

**Property images point to old `fewog.de` CDN — external dependency:**
- 50 `imageUrl` fields in `src/lib/data.ts` point to `https://www.fewog.de/fileadmin/_processed_/...`. These images are served from the legacy site; if that domain changes, redirects, or is decommissioned, all property images break silently.
- Approximately 35 of the 50 entries use the placeholder `csm_dummy_c2f4919c03.jpg` — meaning only ~15 properties have real photos.

**`lang="en"` in the HTML root — wrong language declaration:**
- `src/app/layout.tsx` line 26: `<html lang="en">`. The site is entirely in German. This breaks screen readers, browser translation prompts, and WCAG 2.1 Success Criterion 3.1.1.
- Fix: change to `<html lang="de">`.

**Page `<title>` and `<meta description>` are Next.js boilerplate:**
- `src/app/layout.tsx` lines 15–18: `title: "Create Next App"`, `description: "Generated by create next app"`. These will appear in Google search results and browser tabs for the live site.

**`window.location.href` navigation on home page:**
- `src/app/page.tsx` lines 16–25 uses a `useEffect` + `useState("page")` pattern to trigger `window.location.href` redirects. This causes a hard browser reload instead of a Next.js client-side navigation. It also creates an unnecessary render cycle on every state change.
- The `Nav` component correctly uses `useRouter()`. The home page should do the same.

**"Mitglied werden" CTA has no destination:**
- `src/app/page.tsx` line 53: `<button className="btn btn-ghost btn-arrow">Mitglied werden</button>` — no `onClick` handler.
- `src/components/nav.tsx` line 60: `<button className="nav-cta" onClick={() => go("service")}>Mitglied werden</button>` — routes to the service page, which is not a membership signup page.
- No membership application flow exists anywhere in the codebase.

**`sanity-plugin-media` not in `package.json`:**
- CLAUDE.md lists `sanity-plugin-media` as a required dependency for asset management. It is absent from `package.json`.
- Files: `fewog-app/package.json`

## Medium Priority (should address in v1)

**All pages are `'use client'` — no server-side rendering in use:**
- Every page (`page.tsx`, `wohnen/page.tsx`, `ueberuns/page.tsx`, `aktuelles/page.tsx`, `service/page.tsx`) begins with `'use client'`. This prevents ISR, server components, and the `defineLive` pattern that CLAUDE.md specifies as the default for all content pages.
- Once Sanity is connected, pages will need to be converted to server components with client islands only where interactivity requires it.

**`Footer` component has inconsistent props contract:**
- `src/components/footer.tsx` requires `setPage` prop (line 9).
- `src/app/aktuelles/page.tsx` line 71, `src/app/ueberuns/page.tsx` line 123, and `src/app/service/page.tsx` line 156 all call `<Footer />` without passing `setPage`. TypeScript should catch this but it also reveals the footer's navigation pattern is not properly integrated across all pages.

**Footer navigation links have no `href` attributes:**
- `src/components/footer.tsx` lines 26–43: all footer nav links (`Mitgliedschaft`, `Genossenschaftsprinzip`, `Neubauprojekte`, `Schaden melden`, `Mieter-Magazin`, `Downloads`, `Mietertreff`) are bare `<a>` tags with only `onClick` handlers using `setPage`. They have no `href`, so right-click > open in new tab, copy link address, and keyboard navigation all fail.
- These links should use Next.js `<Link href="...">` components.

**Contact information in `contact-strip.tsx` and `footer.tsx` is placeholder data:**
- Phone number `+49 711 — 58 12 34` and email `kontakt@fewog-fellbach.de` in `src/components/contact-strip.tsx` and `src/components/footer.tsx` differ from the real contact information (`0711/578815-0`, `info@fewog.de`) found in `src/app/aktuelles/page.tsx`. Data is inconsistent across files.
- The address `Bahnhofstraße 12` appears unverified — the business address needs confirmation before go-live.

**Statistic "Jahre Fellbach" uses hardcoded current year:**
- `src/app/page.tsx` line 67: `{2026 - FEWOG_DATA.meta.founded}`. The year `2026` is hardcoded. After January 1, 2027, this will be wrong without a code change.
- Fix: use `new Date().getFullYear() - FEWOG_DATA.meta.founded`.

**`ueberuns/page.tsx` member count conflicts with `data.ts`:**
- `src/app/ueberuns/page.tsx` line 45 states "Die FEWOG besteht aus rund 1.200 Mitgliedern." The authoritative figure in `src/lib/data.ts` (line 30) is `members: 1480`. These two hardcoded values are inconsistent and both wrong once CMS data is in place.

**No `next/image` usage — images are raw `<img>` tags:**
- `src/app/page.tsx` (line 74) and `src/app/wohnen/page.tsx` (lines 68–69) use plain `<img>` tags. Next.js `<Image>` component provides automatic WebP conversion, lazy loading, layout shift prevention (CLS), and responsive sizes. These are important for a property-heavy site.
- `next.config.ts` has no `images` configuration to enable Sanity CDN as an allowed host.

**`next.config.ts` is empty:**
- `src/../next.config.ts` has no configuration at all. Before Sanity images can be served via `next/image`, remote patterns for `cdn.sanity.io` and `www.fewog.de` must be added.

**No `robots.txt` or `sitemap.xml`:**
- No SEO infrastructure exists. The site will be crawlable by default but without guidance. A sitemap covering the 5 routes would be a one-time addition via `src/app/sitemap.ts`.

**Mobile navigation is not functional:**
- `src/app/globals.css` line 112: `.nav-burger { display: none; }` — the mobile hamburger button is always hidden. The nav has a `.nav-mobile-open` class toggled in `nav.tsx`, but there are no CSS rules for the open state that would show the nav links on mobile.
- The nav links and mobile dropdown need media query rules to work on screens below 960px.

## Low Priority / Nice-to-have

**`Fraunces` and `Montserrat` fonts are referenced in CSS but not loaded:**
- `src/app/globals.css` lines 21–23 reference `Fraunces` and `Montserrat` as CSS font family values, but `src/app/layout.tsx` only loads `Geist` and `Geist_Mono` from `next/font/google`. Fraunces and Montserrat will silently fall back to system fonts.

**`service-tile.tsx` onClick prop declared but never used:**
- `src/components/service-tile.tsx` line 9 declares `onClick?: () => void` in the interface, but the component never wires it to the element. All three `ServiceTile` instances in `src/app/page.tsx` (lines 93–110) pass no `onClick` prop, making all service tiles non-interactive.

**`bestand-row.selected` uses a hardcoded blue tint:**
- `src/app/globals.css` line 438: `.bestand-row.selected { background: rgba(56, 128, 224, 0.08); }` — this blue accent has no connection to the FEWOG design system variables (red/green/sand palette). Should use `var(--c-accent-tint)` or `var(--c-secondary-tint)`.

**No favicon or og:image configured:**
- `src/app/layout.tsx` has the default Next.js `favicon.ico` but no `<meta property="og:image">` or Twitter card metadata. Social sharing will show no preview image.

## Tech Debt

**`src/lib/data.ts` will need full replacement:**
- The 50-property static data array, the `FewogData` / `Property` / `District` TypeScript interfaces, and the meta object are all placeholder data that must be migrated to Sanity schemas (`Liegenschaft`, `Stadtteil`, `Seiteneinstellungen`). The interfaces in `data.ts` should be replaced with Sanity TypeGen output (`sanity.types.ts`).
- Files: `src/lib/data.ts`

**`page` state pattern in each page is unnecessary:**
- Every page component declares `const [page, setPage] = useState('<pagename>')` and passes it to `<Nav>`. The Nav component should derive its active state from `usePathname()` (Next.js App Router hook), eliminating all `page`/`setPage` prop drilling.
- Files: `src/app/page.tsx`, `src/app/wohnen/page.tsx`, `src/app/aktuelles/page.tsx`, `src/app/ueberuns/page.tsx`, `src/app/service/page.tsx`, `src/components/nav.tsx`, `src/components/footer.tsx`

**No error boundaries or loading states:**
- No `loading.tsx` or `error.tsx` files exist. Once async Sanity data fetching is added, users will see blank content on slow connections or fetch failures without these.

**SVG district paths are stylised approximations, not geographic:**
- Comment in `src/lib/data.ts` line 52: `// SVG-Pfad (stilisierter Stadtteil — nicht geografisch exakt)`. If a map feature is ever surfaced publicly, this must be clearly labelled as approximate or replaced with actual GeoJSON.

## Hardcoded Content at Risk

All of the following must be migrated to Sanity CMS before the site can be self-managed by the Geschäftsstelle:

- **50 property records** (`src/lib/data.ts` lines 74–123): street, year, units, rooms, sanierung year, imageUrl for all Liegenschaften. Requires Sanity `Liegenschaft` schema.
- **3 district definitions** (`src/lib/data.ts` lines 46–71): names, taglines, color tokens, SVG paths. Requires Sanity `Stadtteil` schema or embedded in settings.
- **Organisation stats** (`src/lib/data.ts` lines 38–44): units (612), members (1480), properties (50), founded year. Should come from a singleton `Seiteneinstellungen` document.
- **Board members / Vorstand / Aufsichtsrat** (`src/app/ueberuns/page.tsx` lines 81–102): 7 named individuals with roles. Requires Sanity `Teammitglied` schema.
- **News / Aktuelles content** (`src/app/aktuelles/page.tsx` lines 26–66): 4 static content blocks (Mängelmeldung, Wohnungssuchende, METRONA portal, Mietertreff). Requires Sanity `Neuigkeit` schema.
- **Document links** (`src/app/service/page.tsx` lines 66, 85–91, 115): Mietermagazin 2025, Geschäftsbericht 2024, Festschrift, Ferienwohnung Nutzungsbedingungen — all direct PDF URLs to old `fewog.de`. Requires Sanity `Dokument` schema with Sanity-hosted assets.
- **Contact information** (`src/components/contact-strip.tsx`, `src/components/footer.tsx`): phone numbers, email, address, opening hours. Requires `Seiteneinstellungen` singleton.
- **Mietertreff dates and times** (`src/app/service/page.tsx` lines 42–55, `src/app/aktuelles/page.tsx` lines 58–66): two recurring monthly events with locations and times. Should be CMS-driven or at minimum editable from a settings document.
