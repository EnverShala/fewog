# Concerns & Technical Debt

**Analysis Date:** 2026-05-19

---

## Critical (blocks launch)

**Sanity CMS not wired up — all content is hardcoded:**
- Issue: Sanity packages are installed (`sanity ^5.25.0`, `next-sanity ^11.6.13`, `@sanity/image-url ^2.1.1`, `@portabletext/react ^6.2.0`) but zero Sanity configuration files exist in `src/`. No `sanity.config.ts`, no `src/app/studio/[[...tool]]/page.tsx` route, no `src/lib/sanity/client.ts`, no schema files anywhere.
- Files: `fewog-app/package.json` (deps installed), `fewog-app/src/` (nothing Sanity-related present)
- Impact: The primary product requirement — "der Genossenschaftsvorstand kann alle Inhalte selbstständig und ohne Programmierkenntnisse pflegen" — is completely unmet. All 50 property records, all news, all team members, and all contact info are hardcoded TypeScript and can only be changed by a developer.

**No `<SanityLive />` in layout — live content updates impossible even after wiring Sanity:**
- Issue: `src/app/layout.tsx` has no `<SanityLive />` component. Without it, content changes in Studio will never appear on the site without a full redeploy. The `defineLive` + `<SanityLive />` pattern mandated by CLAUDE.md is not implemented anywhere.
- Files: `fewog-app/src/app/layout.tsx`

**Impressum and Datenschutz pages are missing — legal violations:**
- Issue: `src/components/footer.tsx` line 16 renders `<a>Impressum</a> · <a>Datenschutz</a> · <a>AGB</a>` links with no `href` attributes. No corresponding routes exist under `src/app/`. A missing Impressum violates §5 TMG. A missing Datenschutzerklärung violates DSGVO Article 13/14. Both are mandatory for any German public-facing website.
- Files: `fewog-app/src/components/footer.tsx:16`, no `src/app/impressum/`, no `src/app/datenschutz/`

## High (should fix before launch)

**Page `<title>` and `<meta description>` are Next.js boilerplate:**
- Issue: `src/app/layout.tsx` lines 15–18 export `title: "Create Next App"` and `description: "Generated by create next app"`. These strings appear in Google search results, browser tabs, and social sharing previews for the live site.
- Files: `fewog-app/src/app/layout.tsx:15-18`
- Fix: Set `title: "FEWOG Fellbach – Genossenschaftliches Wohnen"` and a proper German description.

**`lang="en"` in the HTML root — wrong language declaration:**
- Issue: `src/app/layout.tsx` line 26 declares `<html lang="en">`. The entire site is in German. This breaks screen readers (wrong text-to-speech voice), browser auto-translation prompts, and violates WCAG 2.1 SC 3.1.1 (Language of Page).
- Files: `fewog-app/src/app/layout.tsx:26`
- Fix: Change to `<html lang="de">`.

**Hero image is an Unsplash stock photo — not FEWOG content:**
- Issue: `src/app/page.tsx` line 71 uses `https://images.unsplash.com/photo-1502672260266-1c1ef2d93688` as the hero image. This is a third-party stock photo with unverified commercial brand license, and it does not depict an actual FEWOG property.
- Files: `fewog-app/src/app/page.tsx:71`
- Fix: Replace with a real FEWOG photograph uploaded to Sanity's media library and served via `next/image`.

**~35 of 50 property images use a legacy `fewog.de` placeholder — external CDN dependency with missing photos:**
- Issue: Property `imageUrl` fields in `src/lib/data.ts` fall into two categories: (1) real photos pointing to `https://www.fewog.de/fileadmin/_processed_/` — an external CDN that will break if the old site is decommissioned; and (2) ~35 entries point to `csm_dummy_c2f4919c03.jpg` — the old site's generic placeholder image. Neither is acceptable for launch.
- Files: `fewog-app/src/lib/data.ts:74-123`

**`window.location.href` navigation on home page causes hard browser reloads:**
- Issue: `src/app/page.tsx` lines 14–26 uses `useState('start')` + `useEffect` watching `page` to fire `window.location.href = '/wohnen'`. This triggers a full page reload rather than Next.js client-side navigation. Pages "service", "ueberuns", and "aktuelles" are silently no-ops (`setPage('start')`) so those nav actions from the homepage do nothing.
- Files: `fewog-app/src/app/page.tsx:14-26`
- Fix: Replace with `useRouter().push(...)` or `<Link>` — matching the pattern already used correctly in `src/components/nav.tsx`.

**Footer legal links have no `href` — dead anchors:**
- Issue: `src/components/footer.tsx` line 16: `<a style={{ display: "inline" }}>Impressum</a> · <a style={{ display: "inline" }}>Datenschutz</a> · <a style={{ display: "inline" }}>AGB</a>`. No `href` attribute. These are unclickable; keyboard users cannot activate them, and crawlers see no link target.
- Files: `fewog-app/src/components/footer.tsx:16`

**Nav links are `<a>` with no `href` — inaccessible to keyboard and screen readers:**
- Issue: `src/components/nav.tsx` lines 53–55: nav links are `<a>` elements with `onClick` but no `href`. Users navigating by keyboard (Tab + Enter) cannot activate these links; screen readers announce them as non-interactive. Also breaks right-click "Open in new tab" and browser history prefetching.
- Files: `fewog-app/src/components/nav.tsx:53-55`
- Fix: Use Next.js `<Link href="/wohnen">` etc.

**`sanity-plugin-media` not in `package.json` — CLAUDE.md requires it:**
- Issue: CLAUDE.md specifies `sanity-plugin-media` as a required dependency for asset management in Studio. It is absent from `fewog-app/package.json`. Without it, the Geschäftsstelle staff will have no media browser in Sanity Studio.
- Files: `fewog-app/package.json`

## Medium (fix when time permits)

**All pages are `'use client'` — server components not used:**
- Issue: Every route file (`page.tsx`, `wohnen/page.tsx`, `ueberuns/page.tsx`, `aktuelles/page.tsx`, `service/page.tsx`) declares `'use client'` at the top. This disables server-side rendering, Next.js ISR, and the `defineLive` data-fetching pattern required by CLAUDE.md for all content pages.
- Files: all files under `fewog-app/src/app/`
- Fix: Convert to async server components; isolate interactive islands (`WohnenList`, `Nav`) as separate client components.

**`page`/`setPage` state pattern is redundant and brittle:**
- Issue: Each page declares `const [page, setPage] = useState('<pagename>')` and passes it to `<Nav page={page} setPage={setPage}>`. The Nav already uses `useRouter()` internally. The active page highlight should derive from `usePathname()`, not from a manually maintained string. The current pattern means the active nav item is wrong on direct URL load for all pages.
- Files: `fewog-app/src/app/page.tsx`, `fewog-app/src/app/wohnen/page.tsx`, `fewog-app/src/app/aktuelles/page.tsx`, `fewog-app/src/app/ueberuns/page.tsx`, `fewog-app/src/app/service/page.tsx`, `fewog-app/src/components/nav.tsx`

**Member count is inconsistent across two hardcoded locations:**
- Issue: `src/lib/data.ts` line 30 has `members: 1480`. `src/app/ueberuns/page.tsx` line 45 hardcodes "rund 1.200 Mitgliedern" in German prose. These two numbers contradict each other and neither will update when the real figure changes.
- Files: `fewog-app/src/lib/data.ts:30`, `fewog-app/src/app/ueberuns/page.tsx:45`

**`meta.properties` in data.ts says 50 but CLAUDE.md and old design say 27:**
- Issue: `src/lib/data.ts` line 32 declares `properties: 50`. The home page (`page.tsx` line 44) renders this as "50 Häusern". CLAUDE.md and the original `Design/data.js` prototype both state 27 Liegenschaften. The value displayed publicly is almost certainly wrong.
- Files: `fewog-app/src/lib/data.ts:32`, `fewog-app/src/app/page.tsx:44`, `fewog-app/Design/data.js:9`

**"Jahre Fellbach" stat uses hardcoded year 2026:**
- Issue: `src/app/page.tsx` line 65: `{2026 - FEWOG_DATA.meta.founded}`. After 31 December 2026 this will display the wrong number without a code change.
- Files: `fewog-app/src/app/page.tsx:65`
- Fix: `new Date().getFullYear() - FEWOG_DATA.meta.founded`

**No `next/image` — raw `<img>` tags throughout:**
- Issue: `src/app/page.tsx` line 71 (hero) and `src/app/wohnen/page.tsx` line 117 (property detail) use plain `<img>` tags. `next/image` provides automatic WebP, lazy loading, and prevents Cumulative Layout Shift. `next.config.ts` has no remote pattern config for Sanity CDN or `fewog.de`.
- Files: `fewog-app/src/app/page.tsx:71`, `fewog-app/src/app/wohnen/page.tsx:117`, `fewog-app/next.config.ts`

**`next.config.ts` is completely empty:**
- Issue: `fewog-app/next.config.ts` contains only a blank `nextConfig` object with a comment placeholder. No image remote patterns for `cdn.sanity.io` or `www.fewog.de`, no headers, no redirects. Blocking `next/image` usage with Sanity images.
- Files: `fewog-app/next.config.ts`

**No loading states or error boundaries:**
- Issue: No `loading.tsx` or `error.tsx` files exist anywhere under `src/app/`. Once async Sanity fetches are added, users on slow connections or facing fetch failures will see blank content with no feedback.

**`ServiceTile` component's `onClick` prop is never used:**
- Issue: `src/components/service-tile.tsx` declares `onClick?: () => void` in the `ServiceTileProps` interface but the rendered `<div>` never attaches it as an event handler (the prop is destructured but not spread). The "Schaden melden" tile in `src/app/page.tsx` line 94 passes an `onClick` that silently does nothing.
- Files: `fewog-app/src/components/service-tile.tsx:13,29`, `fewog-app/src/app/page.tsx:94`

**Document links in `/service` and `/ueberuns` point to old `fewog.de` PDFs — external dependency:**
- Issue: Multiple links in `src/app/service/page.tsx` (lines 66, 71, 85, 90, 95, 115) and `src/app/ueberuns/page.tsx` (line 112) point to PDF files hosted on `www.fewog.de/fileadmin/`. If the legacy site is retired these links break. PDFs should be uploaded to Sanity assets or the Vercel-hosted Next.js app.
- Files: `fewog-app/src/app/service/page.tsx`, `fewog-app/src/app/ueberuns/page.tsx`

**No skip-to-content link — WCAG 2.1 SC 2.4.1 violation:**
- Issue: No skip navigation link exists anywhere in the layout. Keyboard-only users must tab through the entire nav on every page before reaching main content.
- Files: `fewog-app/src/app/layout.tsx`

**No `robots.txt` or `sitemap.xml`:**
- Issue: No SEO files exist. No `src/app/robots.ts` or `src/app/sitemap.ts`. The site has 5 routes that should all be listed.
- Fix: Add `src/app/sitemap.ts` (Next.js built-in) returning the 5 page URLs.

**No `og:image` or Twitter/OpenGraph metadata:**
- Issue: `src/app/layout.tsx` exports only `title` and `description`. No `openGraph`, no `twitter` metadata. Social sharing will render no preview image.
- Files: `fewog-app/src/app/layout.tsx:15-18`

**`bestand-row.selected` uses an off-brand blue tint:**
- Issue: `src/app/globals.css` line 495: `.bestand-row.selected { background: rgba(56, 128, 224, 0.08); }`. This blue colour has no relationship to the FEWOG design system variables (Kappelberg-Rot, Weinlaub-Grün, sand palette).
- Files: `fewog-app/src/app/globals.css:495`
- Fix: Replace with `var(--c-accent-tint)` or `var(--c-secondary-tint)`.
